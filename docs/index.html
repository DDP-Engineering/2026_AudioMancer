Translate books and prepare audiobook voices in one desktop app.

AudioMancer helps you take a book from source text to translated text, narrator and character voices, speech blocks, generated audio, and final WAV audiobook files. Each step can be helped by AI or handled manually by the user.

What AudioMancer does.

AudioMancer is made for long book projects. It keeps the text, translation, characters, voices, speech blocks, generated audio, and final exports together in one project folder.

T

Book text workflow

Import a book, extract the text, clean it, split it into useful parts, and export reviewed text files.

L

Translation workflow

Translate a book into the selected project language. The translation can be AI-assisted and then reviewed by you.

V

Voice workflow

Prepare one narrator voice, or prepare narrator plus character voices for dialogue-heavy books.

AI help is optional.

AudioMancer is not an all-or-nothing AI tool. You can use AI for the repetitive work, then review and edit the result. You can also do the same steps manually when accuracy or style matters more than speed.

A

AI-assisted steps

Use AI for cleanup, chunking, translation, character discovery, and speech block creation.

M

Manual review

Edit cleaned text, chunks, translations, characters, voices, and speech blocks before moving forward.

O

Local project files

Projects, logs, generated audio, and final WAV files are stored on your computer.

Install AudioMancer on Windows.

The current public package is tested on Windows. It is portable: download it, extract it, and start the app.

Open Release Page Download SHA256SUMS.txt

Windows steps

  1. Open the GitHub Release page.
  2. Download AudioMancer_Windows_x64.zip.
  3. Download SHA256SUMS.txt if you want to verify the file.
  4. Extract the ZIP to a normal folder, for example C:\AudioMancer.
  5. Start the app with AudioMancer.exe or Start-AudioMancer.cmd.
  6. In the app, create a new project and choose the book language, output language, and voice mode.

Install VoiceWeaver for voice generation.

AudioMancer uses VoiceWeaver as the local voice engine. VoiceWeaver must be running before AudioMancer can generate narrator samples, character samples, or audio blocks.

The current VoiceWeaver package links are maintained on the AudioMancer v1.3.1 release page. Use that release page for both AudioMancer and VoiceWeaver downloads.

VoiceWeaver steps

  1. Open the AudioMancer v1.3.1 release.
  2. Download all 15 VoiceWeaver package parts linked in the VoiceWeaver runtime section.
  3. Put all .partNNN files in the same folder.
  4. Join the parts into one VoiceWeaver ZIP with the PowerShell command below.
  5. Extract the VoiceWeaver ZIP to a folder, for example C:\VoiceWeaver.
  6. Start VoiceWeaver with its Windows start file.
  7. Check that the local address works: http://127.0.0.1:8059/v1/health.

Connect AudioMancer to VoiceWeaver.

After VoiceWeaver is running, tell AudioMancer where to find it. The default local address is http://127.0.0.1:8059.

  1. Start VoiceWeaver first.
  2. Start AudioMancer.
  3. Open TTS Settings from the AudioMancer toolbar.
  4. Set Base URL to http://127.0.0.1:8059.
  5. Leave API Key empty for normal local use.
  6. Press Test Connection.
  7. Check that Qwen3-TTS Base and Qwen3 Voice Design are found.
  8. Save the TTS settings.

Join the VoiceWeaver parts

$outputZip = "VoiceWeaver-v1.5.0-windows-x64-cuda.zip"
$parts = Get-ChildItem "$outputZip.part*" | Sort-Object Name
$out = [IO.File]::Create($outputZip)
foreach ($part in $parts) {
    $in = [IO.File]::OpenRead($part.FullName)
    $in.CopyTo($out)
    $in.Dispose()
}
$out.Dispose()
Get-FileHash -Algorithm SHA256 $outputZip
The expected joined VoiceWeaver ZIP SHA256 is 393cdf6f57907d8b6651b4bf2069b8a19956b6d4830d568c0b5c0a06b0054aa2. Use the checksum information on the v1.3.1 release page to verify the VoiceWeaver package.

Set up AI models.

AudioMancer can use local AI models on your computer or online AI models from an API provider. The same app workflows can also be reviewed and corrected manually.

AI profiles are used for cleanup, chunking, translation, character discovery, and speech blocks. VoiceWeaver is separate and is used for voice generation.

Local AI with LM Studio

  1. Install LM Studio on Windows.
  2. Download and load a chat model in LM Studio.
  3. Start the LM Studio local server.
  4. In AudioMancer, open AI Settings.
  5. Create or edit a profile.
  6. Set Backend Kind to lmstudio.
  7. Set Base URL to http://localhost:1234/v1.
  8. Set Model Name to the model name shown by LM Studio.
  9. Press the connection or model refresh button, then save the profile.
  10. Select that profile as the default for the workflow steps you want to run locally.

Online AI with an OpenAI-compatible API

  1. Get an API key from your AI provider.
  2. In AudioMancer, open AI Settings.
  3. Create or edit a profile.
  4. Set Backend Kind to openai_compatible.
  5. Set Base URL to the provider URL, for example https://api.openai.com/v1.
  6. Paste the API key into API Key.
  7. Set the model name, context window, output tokens, timeout, and temperature.
  8. Test or refresh the profile, then save it.
  9. Select that profile as the default for cleanup, chunking, translation, character discovery, or speech blocks.

Recommended model use.

Use stronger models for translation and character discovery. Use cheaper or smaller models for cleanup and chunking if the result will be reviewed. Keep temperature low for structured steps so the answers are easier to check.

  • Cleanup: fixes extraction and formatting problems.
  • Chunking: splits the book into useful pieces.
  • Translation: translates the book into the selected language.
  • Character Discovery: finds characters and prepares voice notes.
  • Speech Blocks: turns reviewed text into narrator and character lines.
